# 著者FAQ

## 使用しているSTACKのバージョンは？

お使いのサーバー上の STACKドキュメントのトップページにアクセスすると、ページ下部にSTACKプラグインのバージョン番号が表示されます。もしお使いのサイトが`https://maths.assessment/site`なら、STACKドキュメントのURLはおそらく`https://maths.assessment/site/question/type/stack/doc/doc.php`でしょう。ドキュメントはソースコードと共に配布されています。そのため、（最新リリースについて記述された `https://docs.stack-assessment.org/` のドキュメントではなく）このローカルのドキュメントを参照することで、お使いのサイトで実際に利用可能なSTACKの機能を確認することができます。

バージョン番号はすべてのMoodleプラグインで使用される形式 (例 `2022052300`) で表示されます。

## LaTeXとは何ですか?LaTeXを学ぶ手助けはどこにありますか？

LaTeXは文書作成システムです。STACKではいくつかの単純なLaTeXしか必要としませんので、身構える必要はありません。STACKは数式の記述（マークアップ）にのみLaTeXを実際に使用しており、文書構造用のタグは使用しません。

* LaTeXがまったく初めての人のための入門書が[ここ](http://www.andy-roberts.net/misc/latex/index.html)にあります。
* 数学の環境については[こちら](http://www.andy-roberts.net/writing/latex/mathematics_1)を参照してください。
* LaTeXの詳細は<http://www.latex-project.org/guides/>から入手できます。

なお、複雑なLateXはSTACKでは動作しないものもありますのでご注意ください。記述はシンプルに保つようにしてください。

## CASが有効なテキストにHTMLを追加できますか？

HTMLタグは通常通り使用できます。  例えば、これらのタグを使って画像などを挿入することができます。

ただし、JSコードを含む任意の `<script>` タグは記述しないでください。将来のバージョンのSTACKでは、クロスサイトスクリプティングはサポートされなくなります。

## STACKで特定のMaximaの関数が動作しません

セキュリティ上の理由から、すべてのMaxima関数がSTACKで有効になっているわけではありません。
使用しようとしている関数は、STACK がデフォルトでロードしないライブラリに属しているかもしれません。もしその関数の使用にMaximaの load コマンドが必要な場合は、システム管理者または開発者に連絡し、そのライブラリを利用できるように（ロードコマンドを追加するよう）依頼してください。

一部のライブラリはオプションであり、お使いのローカル環境にはインストールされていない可能性もあります。

また、STACKには標準のMaxima関数ではない、STACK独自の関数が多数定義されていることにも注意してください。必要なコマンドが有効になっていないのは、代わりにSTACKが提供する機能を使用すべきだから、という可能性も十分にあります。

## 生徒の解答が完全に簡略化されていることを確認するには？

STACKの設計思想は、学生の解答が持つ「性質」を特定することにあります。「簡約化（Simplify）」とは曖昧な概念です。例えば、\(1\) は \(2^0\) よりも単純ですが、\(2^{2^{10}}\) はおそらく、それが表す整数を10進数で書き下すよりも単純でしょう。\(x+2\) が \(\frac{x^2-4}{x-2}\) よりも単純であることには誰もが同意するでしょうが、以下の式については、最初の式の方が単純かもしれません。

\[ \frac{x^{12}-1}{x-1} =  x^{11}+x^{10}+x^9+x^8+x^7+x^6+x^5+x^4+x^3+x^2+x+1.\]

簡約化とは、暗黙のうちに「等価な最も短い式」を意味すると解釈されがちですが、このことについてはあまり議論されていません。

これらの問題を回避するために、STACKでは教師が求める性質を明示することを期待しています。例えば、因数分解された形を求めるのであれば、「簡約化された形」と表現するのではなく、「因数分解をした形」と表現するべきです。

STACKにおいて非常に有用なテストは、加算と乗算という基本的な算術演算の結合法則と交換法則を認めた上での[等価性判定](Answer_Tests/index ja.md#EqualComAss)です。多くの場合、教師がこのケースで必要としているのはこれです。

## なぜMaximaは `int(1/x,x)=log(abs(x))` と出力しないのですか？

\( \int \frac{1}{x}dx = \log(|x|) \) は、負または正の実軸上では問題ありませんが、複素平面上では正しくないからです。ただし、この挙動を制御する設定があります。

    (%i199) integrate(1/x,x)；
    (%o199) log(x)

    (%i200) integrate(1/x,x), logabs : true；
    (%o200) log(abs(x)).

さらに、[積分の解答テスト](Answer_Tests/index ja.md#Int) では、教師は学生の解答として `log(x)` または `log(abs(x))` （あるいはその両方）を正解として受け入れることができます。

## CASから応答がないのはなぜですか？

問題のデバッグは難しい場合があります。私たちは完全なパーサ（構文解析器）を作成していないため、すべてのエラーを捕捉できるわけではありません。万策尽きた場合は、Maximaに送信されている式そのものを詳細に調べる必要があるかもしれません。

これを行うには

    サイト管理 -> プラグイン -> 質問タイプ -> STACK

ここで`CASデバッグ`がチェックされていることを確認してください。

そうすれば、エラーレポートが表示されるはずです。  例として

    サイト管理 -> プラグイン -> 質問タイプ -> STACK -> ヘルスチェック

そこで、Maximaに送られる式の例を見ることができます。  このような式は[STACK-Maxima sandbox](../CAS/STACK-Maxima_sandbox ja.md)にコピーして評価することができます。 ここで返されるエラーは、発生している問題の特定に役立つかもしれません。

問題の原因は通常、構文的に無効な Maximaコマンドを作成しようとしたことにあります。例えば、[a,,b] はMaximaをクラッシュさせます。私たちは完全なパーサを作成していないため、このような構文エラーのすべてがまだ捕捉されるわけではありません。
