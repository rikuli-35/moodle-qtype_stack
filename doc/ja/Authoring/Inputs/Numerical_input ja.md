# 数値入力

これは数値入力タイプの参考ドキュメントです。

この解答欄タイプでは、学生が何らかの数値を入力する必要があります。

* 変数を含む式は無効として拒否されます。

変数名は禁止されていますが、デフォルトでは関数名は禁止されていません。`sin(pi/2)`は数値を表します。学生に実際に`1`と入力させたい場合は、通常の手順に従って禁止ワードに`sin`を追加して禁止する必要があります。

この解答欄タイプは、学生の解答の末尾のゼロを保持します。例えば、`0.00100`は2つの末尾のゼロを保持します。通常の Maxima の処理や「数式」タイプでは、デフォルトでこれらのゼロは削除されます。

注意点として、「浮動小数点の禁止」などの設定は代数入力内のあらゆる数値に適用できますが、「小数点以下n桁を要求する」などの特定のテストは、この入力タイプにおける単一の数値に対してのみ適用可能です。

## オプション

解答欄の「その他のオプション」フィールドには、以下のタグをカンマ区切りリストで入力してください。なお、これらのオプションに問題変数を使用することはできませんので注意してください。

`floatnum`：学生の解答が浮動小数点数であることを要求します。判定には Maxima の floatnump 述語が使用されます。このオプションや他の真偽値オプションを使用する場合は、追加のオプション欄に floatnum:true のようにカンマ区切りで入力します。

`intnum`：学生の解答が明示的な整数であることを要求します。例えば`6`は有効ですが、`2*3`や`12/2`などは無効です。

`rationalnum`：学生の解答が有理数(分数表示)であることを要求します。判定にはSTACKの`rational_numberp`述語が使用されます。ここでは整数は除外される点に注意してください。

`rationalized`：STACKの`rationalized`関数により判定され、学生の解答の分母に無理数（surd）や\(i\)が含まれないことを要求します。

`mindp:n`：学生が小数点以下`n`桁以上を入力することを要求します。

`maxdp:n`：学生が小数点以下`n`桁以下を入力することを要求します。

`minsf:n`：学生が`n`桁以上の有効数字を入力することを要求します。

`maxsf:n`：学生が`n`桁以下の有効数字を入力することを要求します。

小数点以下の桁数と有効数字の両方を同時に指定することはできません（一方が最小、他方が最大であっても指定できません）。

`mindp=maxdp=n`または`minsf=maxsf=n`の場合、学生は正確に`n`桁/位を入力するよう指示されます。

有効数字の数に曖昧さがある場合（例：100の有効数字は1桁か3桁か？）、学生には有利に解釈されます。
