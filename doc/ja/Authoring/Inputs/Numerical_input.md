# 数値入力

これは数値入力タイプの参考ドキュメントである。

この解答欄タイプでは、学生が何らかの数値を入力する必要がある。

* 変数を含む式は無効として拒否される。

変数名は禁止されているが、デフォルトでは関数名は禁止されていない。`sin(pi/2)`は数値を表す。学生に実際に`1`と入力させたい場合は、通常の手順に従って禁止ワードに`sin`を追加して禁止する必要がある。

この解答欄タイプは、学生の解答の末尾のゼロを保持する。例えば、`0.00100`は2つの末尾のゼロを保持する。通常の Maxima の処理や「数式」タイプでは、デフォルトでこれらのゼロは削除される。

注意点として、「浮動小数点の禁止」などの設定は代数入力内のあらゆる数値に適用できるが、「小数点以下n桁を要求する」などの特定のテストは、この入力タイプにおける単一の数値に対してのみ適用可能である。

## オプション

解答欄の「その他のオプション」フィールドには、以下のタグをカンマ区切りリストで入力する。なお、これらのオプションに問題変数を使用することはできない。

`floatnum`：学生の解答が浮動小数点数であることを要求する。判定には Maxima の floatnump 述語が使用される。このオプションや他の真偽値オプションを使用する場合は、追加のオプション欄に floatnum:true のようにカンマ区切りで入力する。

`intnum`：学生の解答が明示的な整数であることを要求する。例えば`6`は有効であるが、`2*3`や`12/2`などは無効である。

`rationalnum`：学生の解答が有理数(分数表示)であることを要求する。判定にはSTACKの`rational_numberp`述語が使用される。ここでは整数は除外される点に注意する。

`rationalized`：STACKの`rationalized`関数により判定され、学生の解答の分母に無理数（surd）や\\(i\\)が含まれないことを要求する。

`mindp:n`：学生が小数点以下`n`桁以上を入力することを要求する。

`maxdp:n`：学生が小数点以下`n`桁以下を入力することを要求する。

`minsf:n`：学生が`n`桁以上の有効数字を入力することを要求する。

`maxsf:n`：学生が`n`桁以下の有効数字を入力することを要求する。

小数点以下の桁数と有効数字の両方を同時に指定することはできない（一方が最小、他方が最大であっても指定できない）。

`mindp=maxdp=n`または`minsf=maxsf=n`の場合、学生は正確に`n`桁/位を入力するよう指示される。

有効数字の数に曖昧さがある場合（例：100の有効数字は1桁か3桁か？）、学生には有利に解釈される。
