# CASText

## はじめに

CASTextは、CAS（数式処理システム）対応のテキストである。CASTextは、LaTeX数式やCASコマンドを埋め込むことができる、単なるHTMLである。埋め込まれたCASコマンドは、問題がユーザーに表示される前に実行される。なお、LaTeXによる数式記述は、単純な構造のみを使用すること。LaTeXの標準機能のうち、サポートされているのはごく一部に限られる。

問題文をはじめ、STACKの問題に含まれるフィールドの多くはCASTextである。

[Basic HTML](http://www.w3schools.com/html/) に関する情報は他の場所で入手可能である。

現在、STACK は LaTeX 自体を処理しない。LaTeX は [MathJax](http://www.mathjax.org/) を使用するなど、さまざまな方法でユーザーのブラウザ上に表示される。LaTeX の使い方がわからない場合は、[作成者 FAQ](Author_FAQ.md) に簡単な例がいくつかある。

* `\( .... \)` 記号で囲まれたものはすべて、通常の LaTeX と同様に インライン数式 として扱われる。
* 対応する `\[` と `\]` で囲まれたものは、新しい行の中央に配置される ディスプレイ数式 として扱われる。
* LaTeX 数学環境を示すために `$...$` や `$$...$$` のようなドル記号を使用することはサポートしていない。詳細については、以下の [通貨](CASText.md#currency) に関する注記、および [MathJax](../../en/Installation/Mathjax.md#delimiters) のページを参照のこと。
* 数式の外では `\\` のような LaTeX テキスト整形機能を使用せず、代わりに HTML バージョンを使用する。
* `{@` と `@}` の区切り文字で囲まれたものはすべて CAS によって評価され、その結果を表す LaTeX に置き換えられる。いくつかの注意点がある。
  * デフォルトでは、これは インライン数式 として表示される。これは LaTeX 記号を使用することに似ているが、`\({@ stuff @}\)` を使用する必要はなく、`{@ stuff @}` だけで十分である点に注意する。
  * 独自の行の中央に配置されるディスプレイ数式にするには、LaTeX と同様に `\[{@ stuff @}\]` を使用する必要がある。
  * `{@ stuff @}` の外側の `{}` 文字は、出力が LaTeX によって単一のグループとして扱われることを保証するために、出力に残される。
* `{#` と `#}` の区切り文字で囲まれたものはすべて CAS によって評価され、その結果を表す Maxima の値に置き換えられる。これは基本的に、他のツールで使用可能な生の値や、値の入力方法の例示に使用される。
* Maxima 変数 `x` が文字列の場合、`{@x@}` は引用符や LaTeX 環境なしで文字列の内容を出力し、`{#x#}` は引用符で囲まれた文字列の内容を出力する。
* リストの括弧なしでカンマ区切りの値が必要な場合は、`{@stack_disp_comma_separate( list )@}` を使用する。この関数は、リストを中括弧なしの引数の文字列表現に変換する。
* 必要に応じて、1つの LaTeX 数式内に複数の CAS 式を含めることができる。例：`\[ \frac{@p@}{@q@} \]`。多くの問題は、表示レベルではなく CAS のレベルで、例えば CAS 内で変数 `p/q` を定義するのが最善であることに注意する。これは設計上の判断であり、個々のケースで効率的に解決するためには経験を必要とする。この例については、[途中計算の表示](../../en/CAS/Matrix.md#Showing_working) を参照のこと。

以下に例を示す。

    {@sin(1/(1+x^2))@} の導関数は
    \[ \frac{\mathrm{d}}{\mathrm{d}x} \sin \left( \frac{1}{x^2+1} \right) = {@diff(sin(1/(1+x^2)),x)@} \]
    これは <code>{#diff(sin(1/(1+x^2)),x)#}</code> と入力できる。

## 変数 {#Variables}

CASTextでは、[問題変数](Variables.md#Question_variables)フィールドで事前に定義された変数を使用できる。

[ポテンシャル・レスポンス・ツリー](Potential_response_trees.md)のフィールド内でCASTextを使用する場合は、フィードバック変数で定義された変数も使用可能である。

CAS 式が簡略化されるかどうかを制御するには、[選択的簡略化](../../en/CAS/Simplification.md#selective-simplification) に関する詳細を参照できる。

## 問題テキスト {#question_text}

問題テキストは、学生が実際に目にするものである。これは以前のバージョンでは「question text」と呼ばれていた。

これは少し修正された形式の CAStext である。学生が問題に解答できるようにするには、問題テキストに [input](../../en/Authoring/Inputs/index.md) を含める必要がある。

[input](../../en/Authoring/Inputs/index.md) を問題に配置するには、学生の答えが割り当てられる [Maxima](../../en/CAS/Maxima_background.md) 変数の名前を次のタグ内に囲んで配置する。学生の答えが変数 `ans1` に割り当てられる場合は、タグ `[[input:ans1]]` を使用する。また、（学生に表示されるかどうかにかかわらず）検証フィードバックの位置を示すために、対応するタグを配置する必要がある：`[[validation:ans1]]`。正当な変数名であれば何でも使用できる。

* 問題が作成されると、これは適切な [input](../../en/Authoring/Inputs/index.md) に置き換えられる。
* 学生が解答すると、この変数名はそれぞれの [ポテンシャル・レスポンス・ツリー](Potential_response_trees.md) で利用可能になる。
* 入力は、問題テキストに適切なタグを追加することで作成および削除される。したがって、タグを削除すると問題から入力も削除されるため注意する。

別のポテンシャル・レスポンス・ツリーを問題に配置するには、適切な名前を選択し、タグ `[[feedback:prt1]]` を追加するだけでよい。

* これらのタグは、必要に応じて適切なフィードバックに置き換えられる。フィードバックを問題テキストに追加すると、小テストの設定に関係なく、STACK 問題によって常に表示されることに注意する。編集フォームの「個別のフィードバック」ブロックにタグを配置することを好む場合もあるであろう。個別のフィードバックの利用可能性は、Moodle の小テスト設定によって制御される。ここでは、フィードバックタグを問題テキスト内の任意の場所（例：特定の入力の隣）に配置できることと、いつ表示されるかを制御することの間に、いくつかの妥協点がある。これは、多くのパートを持つ問題で最も困難である。単一パートの問題については、「個別のフィードバック」ブロックを使用することを推奨する。
* タグは問題テキスト内の任意の場所に移動できる。
* フィードバックタグを LaTeX 数式内に配置 **しないこと**！
* PRT は、問題テキストに適切なタグを追加することで作成および削除される。したがって、タグを削除すると問題から PRT も削除され、作業が失われる可能性があるため注意する。
* 一部のサイトでは大文字と小文字を区別しない（！）データベースを使用しているため、タグが大文字と小文字の違いだけでなく異なるようにする。例えば、問題内で `[[input:ans_m]]` と `[[input:ans_M]]` を同時に使用することは避ける。フィードバックタグについても同様である。ほとんどのデータベースは大文字と小文字を区別するため、この問題についてはチェックを行わない。

## 個別のフィードバック {#Specific_feedback}

Moodle の問題には「個別のフィードバック（specific feedback）」フィールドがあり、これを学生に表示するかどうか、いつ表示するかについての対応する小テスト設定がある。STACK では、通常ここにポテンシャル・レスポンス・ツリー（PRT）、つまりフィードバックアルゴリズムからのフィードバックタグが配置される。

個別のフィードバックは、任意の問題変数および PRT に依存する可能性があるが、入力に直接依存してはならない。

## 全体的なフィードバック / 模範解答 {#General_feedback}

全体的なフィードバック（数学の文脈では通常「模範解答（worked solution）」と呼ばれる）は、学生が問題に解答した後に表示される。学生が行った反応に依存するフィードバックとは異なり、すべての学生に同じ全体的なフィードバックテキストが表示される。

全体的なフィードバックは任意の問題変数に依存する可能性があるが、入力のいずれにも依存してはならない。また、PRT タグを含んでもならない。この設計上の決定は制限的であるが、ポテンシャル・レスポンス・ツリーを通じて行われるべきフィードバックと、問題が展開される前に記述できるこの問題の模範解答とを意図的に分離するものである。

## 問題記述 {#Question_description}

問題記述（Question description）フィールドは、教師が選択したあらゆる目的に使用できる。このフィールドはダッシュボードページにのみ表示され、学生には表示されない。

このフィールドはさまざまな目的に使用できる。

1. チューターを助けるための追加の「模範解答」や関連する数学。
2. 教育的根拠、または設計の議論。
3. 同僚への問題改善への協力依頼。これにより、問題に関するメモが問題自体の内部に残る。
4. チームでの共同作業における変更履歴。問題の編集者には、デザイナー、テクニカルオーサー、レビュアーなどが含まれる場合がある。`[[todo]]` ブロック内のタグは、作業の次の担当者に注意が必要な問題を知らせるのに役立つ。

注意：一括テスター（bulk tester）には、潜在的な問題を示す `[[todo]]` ブロックをこのフィールドに追加するオプションがある。詳細については、[問題作成ワークフロー](../../en/STACK_question_admin/Authoring_workflow.md) を参照のこと。

これは CASText であるため、変数などを含めることができる。ダッシュボードには、生成されたテキスト（変数値を含む）と生の記述フィールド（`[[todo]]` ブロックが残ったまま、削除されない）の両方が表示される。

## CASText と通貨 {#currency}

通貨にドル記号を使用したいと考えるのは一般的であるが、これは数学の区切り文字としてのドル記号の使用と競合する。このため、STACK では数学の区切りにドルを使用することを推奨していない。

通貨にドルを使用する場合は、バックスラッシュで保護する必要がある（`\$`）。そうしないと、CASText の検証に失敗する。

## 事実とヒント

STACK には、事実（fact）とヒント（hint）に使用される組み込みの公式シートがある。標準テキストは [ファクトシート](../../en/Authoring/Question_blocks/Fact_sheets.md) を使用して追加できる。

## CASText 生成関数

CASText 領域に、例えば異なるパラメータを含むあるテキスト（または SVG グラフィック）のいくつかのバージョンなど、反復的なコンテンツのコピーが複数含まれる場合、STACK 関数 `castext` を使用して、問題変数内で CASText 生成関数を定義することが可能である。

例えば、[問題変数](Variables.md) セクション内で、次のように定義する。

    explanation(x,y):=castext("{@x@} を式に代入すると {@y@} になります。");

これは、どの CASText 領域内でも複数回使用できる。

    [[ comment ]] Generated text [[/ comment ]]
    {@explanation(a,b)@}
    {@explanation(c,d)@}

### 注意

* castext の引数は単一のアトミックな文字列でなければならない。参照ではなく、静的な文字列値である必要がある。
* Maximaは文字列内での改行をそのまま認識するため、castext の引数となる文字列の途中で、そのまま改行して記述できる。
* castext 引数内の引用符には注意が必要である。テキスト内の HTML 属性にはシングルクォーテーション `'...'` を使用する。
* 2つの castext オブジェクトは `castext_concat()` で結合できる。
