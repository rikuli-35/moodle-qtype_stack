# 不等式

非厳密不等式は、それぞれ対応する構文を持つ中置演算子として作成される。

    >=, <=

Maximaでは、$x-1>y$のような単一の不等式だけでなく、$x>1 \text{ and } x<=5$のように論理演算子で結ばれた不等式も対応している。

2つの不等式が同じかどうかは、代数的等価性テストを使って判定できる。 

連鎖した不等式（$1\leq x \leq 2$など）は使用できない。この場合は、$x \geq 1 \text{ and } x\leq 2$のように論理結合子を使う必要がある。
また、学生の解答では`and`と`or`は`nounand`と`nounor`に変換されるため、教師側の解答でもこれらの形式を使わなければならない場合がある。詳細は[命題論理](../Topics/Propositional_Logic.md)を参照すること。

バージョン3.6以降では、Maximaにおける不等式（特に1変数の実数不等式）の対応が大幅に改善された。

# 不等式を扱うための関数

* `ineqprepare(ex)`

この関数は、不等式を `ex>0` または `ex>=0` の形に変換する。このとき、`ex`は常に簡約される。これは代数的等価性の解答テストを念頭に置いて設計されている。

* `ineqorder(ex)`

この関数は式を受け取り、`ineqprepare()`を適用した後で、各部分を並べ替える。例えば、

     ineqorder(x>1 and x<5)；

は、

      5-x > 0 and x-1 > 0

を返す。また、重複する不等式も削除される。不等式の形を構文レベルで整理することで、比較的厳密な形の等価判定を確立している。ここでは、論理積`and`と論理話`or`の可換性・結合性を重視し、不等式の連なりに対して`not`を適用する場合にも対応している。

代数的等価性テストが不等式または不等式の連立を検出した場合、この関数が自動的に適用される。

* `make_less_ineq(ex)`

この関数は不等式の向きを反転させ、常に`A<B` または `A<=B` の形に揃える。それ以外の変形は行わない。  
可換性・結合性まで考慮した等価判定では、不等号の向きの統一変換が自動で行われないため有用である。不等式の向きは揃えておくのが良い。使用方法については、EqualComAssテストの例を参照すること。
