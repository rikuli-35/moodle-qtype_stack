# STACKにおけるドラッグ＆ドロップ問題

ドラッグ＆ドロップ問題は、しばしば「パーソンズ問題（Parson's problems）」と呼ばれる。

ドラッグ＆ドロップ機能は、`[[parsons ..]]` [問題ブロック](Question_block.md)を使って追加できる。
STACK v4.6.0では、`parsons` ブロックに3つの主要な構成があり（それぞれがさらなるカスタマイズをサポートしている）、ブロックヘッダのパラメータ `columns` と `rows` を適切に設定することで実現できる。

1. **証明問題**（使用例：`[[parsons]] ... [[/parsons]]`）：STACK v4.5.0で導入された。詳細なガイドは[パーソンズ問題](Parsons.md)を参照できる。
2. **グルーピング**（使用例：`[[parsons columns="3"]] ... [[/parsons]]`）：
指定した数の列を設定する。各列は**証明問題**構成の左側の列と同様に動作し、学生は各列の先頭からアイテムをドラッグ＆ドロップできる。
アイテムをグループ分けすることだけが重要で、行の位置が問題にならない場合や、各列の長さが可変の場合に便利である。例題は[グルーピング](Grouping.md)のページにある。
3. **グリッド**（使用例：`[[parsons columns="3" rows="2"]] ... [[/parsons]]`）：
`(columns, rows)` の形状のグリッドを設定する。学生はグリッド内の任意の位置にアイテムをドラッグ＆ドロップできる。例題は[グリッド](Grid.md)のページにある。

多くの**グリッド**形式の問題は、**グルーピング**の設定でも作成できることに注意する。
両者の主な違いは、**グリッド**ではグリッド内の任意の位置にアイテムをドラッグでき、上のセルが埋まっているかどうかに関係なく配置できるのに対し、**グルーピング**では各列内のリストの末尾にのみアイテムをドラッグできるという点である。

[トラブルシューティング](Troubleshooting.md)に関する別途のドキュメントもある。
