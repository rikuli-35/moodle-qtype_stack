# STACKにおけるGeoGebra

著者：Tim Lutz - エディンバラ大学およびハイデルベルク教育大学、2022-23年。

STACKは、GeoGebra（[https://geogebra.org](https://geogebra.org)）を使用した動的グラフィックスの組み込みをサポートしている。このページは、STACK問題の一部としてGeoGebraワークシートを表示する方法と、GeoGebraワークシートをSTACKの解答欄に接続する方法に関するリファレンスドキュメントである。

評価を支援するために、STACKは[幾何関連のMaxima関数](../../CAS/Geometry.md)を多数提供している。

GeoGebra自体の内部でもかなりの計算や処理を行うことが可能である。多くの場合、数学的な性質をGeoGebraアプレット内で確認し、その**結果**をSTACKの解答欄に連携させる方がはるかに効率的である。このような結果には、関連するオブジェクト間の距離やブール値の結果などが含まれる。

STACKの設計上の現在の制約として、問題変数内で解答欄の名前と同じ変数名を使用することはできない。
例えば、「set」命令で点 \\(A\\) の初期位置をランダムに生成しつつ、同時に「watch」命令でこのGeoGebraオブジェクトを解答欄 `input:A` に連携させることはできない。
このような場合には、GeoGebra内に（おそらく非表示の）**依存オブジェクト**を作成し、それを解答欄に対応させる必要がある。
（これはSTACKの設計に組み込まれたものであり、変更することはできない。）
