## 問題を探す

STACKの主な目的は、教師が自分自身の教材を作成できるようにすることです。とはいえ、STACKには技術的な部分もあるため、既存の問題をテンプレートとして利用し、ニーズに合わせて修正することから始めるのが理にかなっている場合も少なくありません。STACKの教材は、以下の方法で利用可能です。

1. STACK question library：すべての教師がすぐに使用できます。
2. 教材バンク (Moodle .xml) :インポートして利用できます。
3. [gitsync](https://github.com/maths/moodle-qbank_gitsync)プラグイン : このプラグインを通じて配布される教材バンクです。

これまでのクイックスタートガイドに従ってきたのであれば、自分自身の問題をゼロから作成する方法もすでに理解しているはずです。

### 1. Stack question library

Stack question libraryは、すべての教師がすぐに利用可能です。

1. 新しいSTACK問題を作成します。
2. 空白のSTACK問題から、「Stack question library」へのリンクをクリックします。
3. 問題を選択し、プレビューを確認します。
4. 問題を現在の問題バンクに直接インポートします。（例：新しい問題を作成したカテゴリ）
5. 問題バンクに戻ります。インポートした問題は、確認/編集/使用可能になります。

例えば、 `Doc-Examples\Reveal_block_example.xml` をインポートしてみてください。この問題は、複数の入力の使用方法と「reveal」ブロックのデモを提供します。

### 2. 既存のサーバーから問題をインポートする

既存のサーバーから問題をインポートする方法を説明します。

まず、既存の問題をエクスポートする必要があります。

1. 問題をエクスポートしたいMoodleサーバー上のモジュールにログインし、管理ブロックの `問題バンク` をクリックします。次に `問題`プルダウン下部の`エクスポート`をクリックします。
2. `Moodle XMLフォーマット` をクリックし、エクスポートしたいカテゴリを選択します。Moodleでは、個々のカテゴリ単位でしかエクスポートできません。
3. `問題をファイルに問題をエクスポートする` をクリックします。これにより、そのカテゴリ内のすべての問題が含まれたファイルがダウンロードされます。

これらの問題をコースにインポートするには：

1. 問題をエクスポートしたいMoodleサーバー上のモジュールにログインし、管理ブロックの `問題バンク` をクリックします。次に `問題`プルダウン下部の`インポート`をクリックします。
2. `Moodle XMLフォーマット` をクリックしてから、先にダウンロードした ?.xml ファイルをドラッグ＆ドロップし、インポート、続いて `続ける` をクリックします。これにより、モジュールの問題バンクに問題のコピーが表示され、必要に応じて自由に変更できるようになります。

### 3. gitsync module

[gitsync モジュール](https://github.com/maths/moodle-qbank_gitsync)の目的は、Moodle の問題バンクの一部を外部ファイルシステムと同期することです。これは高度な機能であり、追加のサーバー設定が必要です。

主な使用ケースは、（i） 単一の Moodle サイト内の複数のコース間で問題を共有すること、（ii） 複数のサイト間で問題を共有することです。通常、外部ファイルシステムはバージョン管理リポジトリ（例：git）の一部となっています。そのため、外部ファイルシステムに対してバージョン管理ツールを使用することで、バージョン間の差分の追跡、変更のマージ、ブランチを使用した異なるバージョンの維持などを行うことができます。
